# Backend Scope

This backend is for one product: an ecommerce website for selling medical equipment.

Keep the backend focused on the buying and selling flow only.

## In scope

- product catalog
- product categories
- product details
- product stock availability
- buyer inquiries
- vendor or seller profile data needed for listings
- admin actions needed to manage the marketplace safely
- API health checks
- demo seed data for the medical equipment marketplace

## Out of scope

Do not add features that are not needed for the ecommerce flow, including:

- KPI modules
- analytics dashboards
- unrelated reporting
- event planning
- blog or CMS features
- social features
- unrelated workflow tools
- marketplace categories outside medical equipment

## Rule for future backend work

Before adding a backend app, model, route, service, task, or dependency, ask:

Does this directly help customers find, view, inquire about, or buy medical equipment?

If the answer is no, keep it out of the backend.
